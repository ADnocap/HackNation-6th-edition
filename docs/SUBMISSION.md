# Submission Checklist

**Deadline: Sunday, July 19 — 9:00 AM ET / 3:00 PM Paris. Freeze features by ~1:00 PM Paris.**

Exact deliverables will be confirmed with the challenge reveal; this list covers what Hack-Nation submissions have required in past editions. Update once the official portal/form is announced.

## Deliverables

- [ ] **Public GitHub repo** — clean README with: what it does, the challenge it addresses, how to run it, team members.
- [ ] **Demo video (~2–3 min)** — screen recording of the working product. Show the real thing, not slides. Upload unlisted to YouTube/Loom as backup.
- [ ] **Short write-up / one-pager** — problem, solution, how it works, what's next (venture angle).
- [ ] **Submission form filled** — link TBD (announced with challenges).

## Pre-submission sanity checks

- [ ] App runs from a clean clone following only the README instructions.
- [ ] No secrets committed (`.env` gitignored, `.env.example` present).
- [ ] Demo happy-path tested three times in a row without failure.
- [ ] Video actually uploaded and link is publicly accessible (test in incognito).
- [ ] Everyone on the team is listed on the submission.

## If we make finals (top 3 of our challenge)

- [ ] 3-minute pitch, Saturday July 25, 12:00–2:00 PM ET (6:00–8:00 PM Paris), public event.
- [ ] Structure: problem (30s) → live demo (90s) → impact & venture potential (45s) → ask/close (15s).
- [ ] Have a pre-recorded demo fallback in case live demo breaks.
